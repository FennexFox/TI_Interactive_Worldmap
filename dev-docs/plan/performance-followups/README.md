# Performance follow-ups

This folder holds performance follow-up drafts that may span multiple implementation issues.

Use this folder for proposed investigations, evidence summaries, and issue-body drafts that are not yet durable project documentation.

When a follow-up becomes an active GitHub issue, copy the durable problem statement and acceptance criteria into the issue. Keep only local planning details here. When the follow-up is completed, merged, or superseded, delete or archive the folder after promoting any still-useful conclusions into `README.md`, `AGENTS.md`, `.github/**`, or the relevant issue discussion.
