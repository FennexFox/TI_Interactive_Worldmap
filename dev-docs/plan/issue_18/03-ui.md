# Phase 03: Scenario selector UI, documentation, E2E, and Pages build

## Goal

- Expose scenario switching to users, document the schema/regeneration flow, validate the static Pages build, and close #18 with E2E evidence.

## Scope

- Add compact scenario selector UI with active summary text.
- Add i18n strings and styling consistent with the existing control panel.
- Add E2E tests for default load, selector options, switching 2022/2070, search/panel/overlay usability, and return to 2026.
- Update README schema/regeneration documentation.
- Rebuild and verify checked-in Pages output.

## Non-goals

- Do not redesign the full sidebar or map UI.
- Do not add scenario comparison, diff views, or arbitrary scenario imports.
- Do not add in-app explanatory text beyond the label/status the issue requested.

## Affected files

- `src/index.html`
- `src/styles.css`
- `src/app.js`
- `tests/*.spec.js` or a new `tests/scenarios.spec.js`
- `README.md`
- Generated by commands only: `docs/index.html`, `docs/assets/**`, `docs/data/**`

## Implementation steps

- Add a scenario control near the existing search/map controls, likely a segmented select or compact native select with options 2022, 2026, and 2070.
- Render a concise summary line from generated scenario summary data, including active scenario and key counts.
- Wire the control to `setActiveScenario(nextScenario)` and update control state after successful switches.
- Add Korean and English strings for labels/options/summary.
- Add E2E tests:
  - app defaults to 2026;
  - selector lists 2022/2026/2070;
  - switching to 2022 updates the active scenario label and leaves search/map usable;
  - switching to 2070 updates the active scenario label and leaves search/map usable;
  - selecting/searching after switch renders overlays without stale highlights;
  - returning to 2026 restores default scenario state.
- Update README to describe schema version 2, `scenario_bundle.generated.json`, and WSL/local-game regeneration commands.
- Run full validation and record manual smoke evidence.

## Acceptance criteria

- Scenario selector is visible, compact, keyboard-accessible, and defaults to 2026.
- Selector options are exactly 2022, 2026, and 2070.
- Active summary text updates when switching scenarios and comes from generated data.
- Search results, selected panel, overlays, incoming claims, warnings, and map colors use active scenario data after switches.
- E2E coverage exercises default and both non-default scenarios.
- README explains how to regenerate all scenarios and clarifies that duplicated geometry is an intermediate schema.
- `npm run build`, `npm run verify`, and `npm run test:e2e` pass after generated output is rebuilt.

## Validation commands

- npm run build
- npm run verify
- npm run test:e2e

## Manual smoke tests

- Default load uses scenario 2026 and exposes 2022/2026/2070 selector options
- Switching to 2022 and 2070 updates active scenario data, search, panels, and overlays without stale selections

## Rollback risks

- UI rollback must also remove or neutralize tests that assume visible selector behavior.
- README and generated Pages output must remain consistent with source code after rollback.
