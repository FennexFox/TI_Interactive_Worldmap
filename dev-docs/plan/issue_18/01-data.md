# Phase 01: Scenario generated data schema, builders, and verifier

## Goal

- Produce reproducible 2022/2026/2070 generated data in a schema the static app can consume, while preserving 2026 as the default and keeping legacy generated files compatible.

## Scope

- Add scenario bundle generation from existing per-scenario region, nation, research, and claim builders.
- Update Pages packaging to include the bundle and default 2026 top-level compatibility data.
- Update generated-output verification for all configured scenarios.
- Add focused unit coverage for bundle/schema behavior where source functions are testable.
- Rebuild generated data from local Terra Invicta templates after source changes.

## Non-goals

- Do not migrate to shared geometry in this phase.
- Do not add frontend scenario switching behavior beyond keeping current 2026 load working.
- Do not hand-edit generated JSON or Pages output.
- Do not refresh Unity region outlines unless current geometry is unusable.

## Affected files

- `tools/build_pages.py`
- `tools/rebuild_pages.py`
- `tools/verify_generated_outputs.py`
- New helper if useful: `tools/build_scenario_bundle.py`
- `scripts/build-wsl.sh`
- `tests/test_catalog_builders.py` or a new focused Python test file
- `README.md` generated-data sections if schema/regeneration commands change
- Generated by commands only: `data/generated/**`, `docs/data/**`, `docs/assets/data.generated.js`

## Implementation steps

- Define constants for supported scenarios `2022`, `2026`, `2070` and default `2026` in generator/tooling code.
- Generate or stage per-scenario outputs under deterministic paths, then assemble `scenario_bundle.generated.json`.
- Ensure top-level default files (`region_map.generated.json`, `claim_map.generated.json`, `nations.catalog.json`, `research.catalog.json`) remain the 2026 scenario.
- Update `build_pages.py` to read/write/copy the scenario bundle and pack `defaultScenario`, `scenarios`, and compatibility top-level data into `window.TI_DATA_PROMISE`.
- Update `rebuild_pages.py` and `scripts/build-wsl.sh` so local-game rebuilds generate all supported scenarios by default while retaining explicit single-scenario behavior only if it is still useful for debugging.
- Extend verifier checks: all scenarios exist, default is 2026, each scenario has regions, nation catalog, claim map, summaries, zero unmatched claim rows, resolved claim targets, resolved project metadata, and docs data matching source generated data.
- Add targeted tests for bundle creation and scenario verifier failures.
- Rebuild generated data and Pages output from the local Templates path.

## Acceptance criteria

- `data/generated/scenario_bundle.generated.json` exists with schema version 2, default scenario 2026, and exactly 2022/2026/2070 scenario entries.
- Each scenario entry contains `regionMap`, `claimMap`, `catalogs.nations`, `catalogs.research`, and summary counts derived from those files.
- Default top-level generated files match the 2026 scenario data used in the bundle.
- `docs/data/scenario_bundle.generated.json` and `docs/assets/data.generated.js` contain the same scenario content as source generated data.
- Verifier fails on missing scenarios, wrong default scenario, unresolved claim targets, or docs/source mismatches.
- No generated file is manually edited.

## Manual smoke tests

- Inspect generated bundle keys/counts with targeted scripts.

## Rollback risks

- Reverting only this phase must restore the single-scenario generated schema and build scripts.
- Generated output size can grow enough to affect page load; record size after rebuild.
- If a scenario has unresolved data, do not paper over it in frontend code.
