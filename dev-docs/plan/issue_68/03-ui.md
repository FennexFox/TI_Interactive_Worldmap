# Phase 03: Extract UI controls panels and i18n

## Goal

- Extract UI text/i18n helpers, side-panel rendering, and control wiring from `src/app.js` while preserving DOM output and user interactions.

## Scope

- Move translation table and localization helpers into `src/ui/i18n.js`.
- Move reusable HTML/formatting helpers and side-panel renderers into `src/ui/panels.js` or focused UI modules.
- Move search/dropdown/filter/scenario/reachable-capital button wiring into `src/ui/controls.js` where practical.

## Non-goals

- Do not redesign layout, copy, accessibility semantics, or CSS.
- Do not change search ranking or dropdown behavior except through behavior-preserving extraction.
- Do not implement new language support beyond preserving current English/Korean behavior.

## Affected files

- `src/app.js`
- New `src/ui/i18n.js`
- New `src/ui/aside-cards.js`
- New `src/ui/panels.js`
- New `src/ui/controls.js`
- `tests/language.spec.js`
- `docs/assets/**` after build
- `tools/build_pages.py`
- `tools/verify_generated_outputs.py`
- `package.json`

## Implementation steps

- Extract translation and formatting primitives.
- Extract panel row/render helpers with injected dependencies.
- Extract event binding functions with callbacks supplied by `app.js`.
- Verify language refresh, search/dropdown, filters, pinned panel, reachable candidates panel, and nation info panel.

## Acceptance criteria

- `app.js` no longer owns the translation table or most panel rendering details.
- Extracted UI modules do not import app state directly; they receive state/model/callback dependencies.
- Search, selection, filters, language, and panels behave as before.

## Validation commands

- npm run build
- npm run verify
- npm run test:e2e

## Manual smoke tests

- Select a country, select a region, clear selection, use search/dropdown, toggle filters, switch language, pin/unpin reachable capital candidate.

## Rollback risks

- Medium-high: UI extraction can subtly change event timing or generated markup. Mitigation: compare selectors covered by existing e2e tests and keep DOM structure unchanged.
